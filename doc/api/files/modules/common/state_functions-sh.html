common.state.updateInfo

Goes through all available modules for the current model and version, and collects vital information in the state DB.

tables that are persistent

  • tasks
  • days
  • installed
  • types (constant)

tables that contain only whats relevant for the current configuration

  • modules
  • metadata
  • instance_tasks (each instance selects its own tasks) (there are others, but they are taken care of in common.task.init)

common.state.storeStatus

Stores the current status of a run, can be either $CXR_STATUS_RUNNING, $CXR_STATUS_SUCCESS or $CXR_STATUS_FAILURE.  In the case of ERROR, we will take note of the fact.

If a task starts, checks if this task was already executed.  The name of the task is determined via common.task.getId if it is not passed in.  The ability to pass a task name is used by installers and tests alike, they use a slightly different approach.

If the user wants to run a specific module, we will disregard the fact that the step already ran, but we advise -F

Returns

$CXR_RET_OK if OK (true) $CXR_RET_ALREADY_RUN if this task was already started (false)

Parameters

$1status ($CXR_STATUS_RUNNING, $CXR_STATUS_SUCCESS or $CXR_STATUS_FAILURE.)
