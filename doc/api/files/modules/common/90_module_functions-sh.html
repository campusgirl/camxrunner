cxr_common_get_module_type

Searches for a module name (not in common and installers) and returns its type.  Module names need to be unique across all module types!  Used by cxr_common_is_dependency_ok.  We look at CXR_PREPROCESSOR_DAILY_INPUT_DIR CXR_PREPROCESSOR_ONCE_INPUT_DIR CXR_POSTPROCESSOR_DAILY_INPUT_DIR CXR_POSTPROCESSOR_ONCE_INPUT_DIR CXR_MODEL_INPUT_DIR

Parameters

$1name of module (without prefix or suffix, just something like “convert output”

cxr_common_run_modules

Calls all or just one single module (adressed by their module name) at a specific point in time (or One-Time) Here, a sequential approach is implied.

Parameters

$1Type of modules to run
[$2]Single Step (number)

cxr_common_process_sequentially

Executes all (needed) modules in sequential order (non-parallel version of the task_functions) If only one process is used, this is faster because there is less overhead.  For each part of the processing, we check if we need to run it and pass any limitations on to cxr_common_run_modules.

test_module

Runs the predefined tests for this module.  If you add or remove tests, please update CXR_META_MODULE_NUM_TESTS in the header!
